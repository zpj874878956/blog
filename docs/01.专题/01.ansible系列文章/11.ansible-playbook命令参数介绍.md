# `ansible-playbook` 命令常用参数及其简化版本：

### 1. `-i`, `--inventory`

- **作用**：指定库存文件，通常是主机清单文件，告诉 Ansible 执行任务时需要连接哪些主机。

- **简化参数**：`-i` 即为简化参数。

- **示例**：
  
  ```bash
  ansible-playbook -i inventory.ini playbook.yml
  ```

### 2. `--ask-pass`, `-k`

- **作用**：提示输入 SSH 密码，用于在目标主机需要密码认证时连接。

- **简化参数**：`-k` 等同于 `--ask-pass`。

- **示例**：
  
  ```bash
  ansible-playbook -k -i inventory.ini playbook.yml
  ```

### 3. `--ask-become-pass`, `-K`

- **作用**：提示输入 `sudo` 密码，用于目标主机需要提权时执行任务。

- **简化参数**：`-K` 等同于 `--ask-become-pass`。

- **示例**：
  
  ```bash
  ansible-playbook -K -i inventory.ini playbook.yml
  ```

### 4. `--become`, `-b`

- **作用**：启用任务提权，表示以超级用户身份执行任务（通常是 `root`）。

- **简化参数**：`-b` 即为简化参数。

- **示例**：
  
  ```bash
  ansible-playbook -b -i inventory.ini playbook.yml
  ```

### 5. `--check`, `-C`

- **作用**：执行 dry-run（干运行），不会对目标主机做实际的更改，只会显示将会执行的任务。

- **简化参数**：`-C` 即为简化参数。

- **示例**：
  
  ```bash
  ansible-playbook -C -i inventory.ini playbook.yml
  ```

### 6. `--diff`

- **作用**：在更改文件时，显示文件差异（diff）。对于修改的配置文件，可以查看具体的变动。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --diff -i inventory.ini playbook.yml
  ```

### 7. `-v`, `--verbose`

- **作用**：增加输出详细信息，显示任务执行的更多细节。可以使用 `-v`（简化）来增加一次详细输出，`-vv` 或 `-vvv` 提供更多级别的详细信息。

- **简化参数**：`-v` 即为简化参数。

- **示例**：
  
  ```bash
  ansible-playbook -v -i inventory.ini playbook.yml
  ```

### 8. `--extra-vars`, `-e`

- **作用**：提供额外的变量，可以在运行时通过命令行传入变量，覆盖 playbook 中的默认值。

- **简化参数**：`-e` 即为简化参数。

- **示例**：
  
  ```bash
  ansible-playbook -e "key=value" -i inventory.ini playbook.yml
  ```

### 9. `--limit`

- **作用**：限制 playbook 只运行在指定的主机或组上，避免在所有目标主机上执行。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --limit webservers -i inventory.ini playbook.yml
  ```

### 10. `--tags`

- **作用**：只执行带有指定标签的任务。用于在执行 playbook 时选择性地运行某些任务。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --tags install -i inventory.ini playbook.yml
  ```

### 11. `--start-at-task`

- **作用**：从指定任务开始执行 playbook。适用于需要从某个任务恢复执行的场景。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --start-at-task="Install packages" -i inventory.ini playbook.yml
  ```

### 12. `--syntax-check`

- **作用**：仅检查 playbook 语法，不执行任何任务。用于调试或验证 playbook 是否存在语法错误。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --syntax-check playbook.yml
  ```

### 13. `--private-key`, `-i`

- **作用**：指定用于 SSH 登录的私钥文件，通常与 `--ask-pass` 配合使用来避免密码输入。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --private-key=path/to/key.pem -i inventory.ini playbook.yml
  ```

### 14. `--force-handlers`

- **作用**：强制执行所有 handlers，即使它们没有被标记为需要执行的状态。

- **简化参数**：没有简化版本。

- **示例**：
  
  ```bash
  ansible-playbook --force-handlers -i inventory.ini playbook.yml
  ```

这些参数可以根据实际需求组合使用，以便更灵活地控制 playbook 的执行。
