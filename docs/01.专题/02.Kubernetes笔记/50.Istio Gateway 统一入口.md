# 🧱 ① 统一 Gateway（核心）

👉 只保留一个 Gateway

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: istio-unified-gateway
  namespace: istio-system
spec:
  gatewayClassName: istio
  listeners:
  - name: http
    port: 80
    protocol: HTTP
    hostname: "*.zzppjj.top"
    allowedRoutes:
      namespaces:
        from: All
```

# 统一 TLS（核心）

## 🎯 目标

所有访问变成：

```text
https://*.zzppjj.top
```

------

# 🔐 第 1 步：创建统一 TLS 证书（自签 or CA）

## ✔ 方式 A（推荐生产）：CA证书

如果你有企业 CA，直接签：

------

## ✔ 方式 B（测试/内网）：自签证书

```bash
openssl req -x509 -nodes -days 365 \
-newkey rsa:2048 \
-keyout tls.key \
-out tls.crt \
-subj "/CN=*.zzppjj.top"
```

------

## 创建 secret

```bash
kubectl create -n istio-system secret tls wildcard-zzppjj-tls \
--key=tls.key \
--cert=tls.crt
```

------

# 🧱 第 2 步：升级 Gateway（HTTP → HTTPS）

👉 替换你的 istio-unified-gateway

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: istio-unified-gateway
  namespace: istio-system
spec:
  gatewayClassName: istio
  listeners:

  # HTTP（自动跳 HTTPS）
  - name: http
    port: 80
    protocol: HTTP
    hostname: "*.zzppjj.top"
    allowedRoutes:
      namespaces:
        from: All

  # HTTPS（核心）
  - name: https
    port: 443
    protocol: HTTPS
    hostname: "*.zzppjj.top"
    tls:
      mode: Terminate
      certificateRefs:
      - name: wildcard-zzppjj-tls
        kind: Secret
    allowedRoutes:
      namespaces:
        from: All
```

------

## 应用

```bash
kubectl apply -f gateway.yaml
```

# 🌐 第 3 步：HTTP 自动跳 HTTPS（推荐）

创建 HTTPRoute（全局跳转）

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: redirect-to-https
  namespace: istio-system
spec:
  parentRefs:
  - name: istio-unified-gateway
  hostnames:
  - "*.zzppjj.top"
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /
    filters:
    - type: RequestRedirect
      requestRedirect:
        scheme: https
        statusCode: 301
```

# 🚀 第一部分：灰度发布

------

# 🎯 目标

同一个服务：

- 90% → stable
- 10% → canary

------

# 📦 1. Service 分版本（关键）

```yaml
apiVersion: v1
kind: Service
metadata:
  name: vmagent-vmagent
  namespace: monitoring
spec:
  selector:
    app: vmagent
  ports:
  - port: 8429
```

------

👉 Pod 打标签：

```bash
kubectl label pod vmagent-xxx version=stable
kubectl label pod vmagent-yyy version=canary
```

------

# 📦 2. 灰度 HTTPRoute（权重分流）

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: vmagent-gray-route
  namespace: monitoring
spec:
  parentRefs:
  - name: istio-unified-gateway
    namespace: istio-system
    sectionName: https

  hostnames:
  - vmagent.zzppjj.top

  rules:
  # 🎯 10% 灰度流量
  - matches:
    - headers:
      - name: x-canary
        value: "true"
    backendRefs:
    - name: vmagent-vmagent
      port: 8429
      weight: 10

  # 🎯 默认 90% stable
  - backendRefs:
    - name: vmagent-vmagent
      port: 8429
      weight: 90
```

## 🧪 测试灰度

# 🚀 第三部分：限流（SRE核心）

# 🧨 1. Envoy 全局限流（IP级）

```yaml
apiVersion: networking.istio.io/v1beta1
kind: EnvoyFilter
metadata:
  name: rate-limit
  namespace: istio-system
spec:
  configPatches:
  - applyTo: HTTP_FILTER
    match:
      context: GATEWAY
    patch:
      operation: INSERT_BEFORE
      value:
        name: envoy.filters.http.ratelimit
```

# 🚀 第四部分：熔断（防止雪崩）

------

```yaml
apiVersion: networking.istio.io/v1beta1
kind: DestinationRule
metadata:
  name: vmagent-circuit-breaker
  namespace: monitoring
spec:
  host: vmagent-vmagent
  trafficPolicy:
    connectionPool:
      tcp:
        maxConnections: 100
      http:
        http1MaxPendingRequests: 100
        maxRequestsPerConnection: 10
    outlierDetection:
      consecutive5xxErrors: 5
      interval: 10s
      baseEjectionTime: 30s
```
