# pod漂移原因分析

## 1、主动漂移

**用户或者控制器操作**

1.手动删除或者重建pod

2.节点被打了污点（drain）

**弹性扩缩容**

1.水平扩缩（HPA）调整副本数

2.集群自动扩缩（CA）

## 2、被动漂移

**节点故障或者资源耗尽**

**调度策略触发**

**存储卷不可用**

# 漂移后pending问题分析

Pod Pending → 检查 `kubectl describe pod` 事件  
              │
              ├─→ 资源不足 → 扩容节点/调整 requests  
              │
              ├─→ 调度策略冲突 → 检查标签、污点、亲和性  
              │
              ├─→ 存储卷问题 → 检查 PVC/PV  
              │
              ├─→ 镜像问题 → 修复镜像拉取配置  
              │
              └─→ 节点或插件异常 → 修复节点/K8s 组件
